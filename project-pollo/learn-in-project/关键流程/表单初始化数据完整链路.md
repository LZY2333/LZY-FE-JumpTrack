# 表单初始化数据完整链路 —— 以 text-to-image (FormGenerate) 为例

> 从页面加载到表单最终呈现给用户，表单数据经历了 **6 个数据源的分层合并**。
> 本文梳理完整的初始化链路、各数据源的触发条件与优先级、以及相关 Context/Store 的协作方式。

---

## 一、总览：6 层数据源与优先级

```
优先级（低 → 高）：

① 硬编码默认值         — FormGenerate 内写死的 initialValues
     ↑ 被展开覆盖
② localStorage 缓存   — useFormLocalValues 读取上次填写的配置
     ↑ 被 setFieldsValue 覆盖
③ URL query → API     — useFormRemoteValues 根据 URL 参数调接口回填
     ↑ 独立通道
④ 隐私设置             — usePrivacySettings 单独处理 published/protectionMode
     ↑ 被 setFieldsValue 覆盖
⑤ 模型落地页中转        — useLocalFormValuesFromModelPage 从 localStorage 读取
     ↑ 被 setFieldsValue 覆盖
⑥ 运行时事件           — globalEventEmitter 驱动的表单回填/重置
```

### 数据源对比表

| # | 数据源 | hook | 存储位置 | 触发时机 | 写入方式 |
|---|--------|------|----------|----------|----------|
| ① | 硬编码默认值 | — | 代码内 | 组件挂载 | `initialValues` prop |
| ② | localStorage 缓存 | `useFormLocalValues` | `localStorage['local-storage-form-data']` | 组件挂载（读）+ 字段变化（写） | `initialValues` prop 展开 |
| ③ | URL → API 远程数据 | `useFormRemoteValues` | 服务端 API | URL 带 recordId/videoId/lora 时 | `form.setFieldsValue()` |
| ④ | 隐私设置 | `usePrivacySettings` | `localStorage['privacy-settings-{userId}']` | 组件挂载 + 订阅变化 | `form.setFieldsValue()` |
| ⑤ | 模型落地页中转 | `useLocalFormValuesFromModelPage` | `localStorage['by-model-page-local-form-values']` | `?customType=modelPage` 时 | `form.setFieldsValue()` |
| ⑥ | 运行时事件 | `useGeneratorFormFill` / `useGeneratorFormReset` | 内存（事件总线） | 用户切换历史记录/新建 | `form.setFieldsValue()` |

## 三、useFormInitialValues 内部合并图

```txt
useFormInitialValues({ formKey, form, omitStorageKeys, postRemoteData })
│
├── useFormLocalValues({ formKey, form, omitStorageKeys })
│   │
│   ├── useFormLocalStorage()
│   │   └── useLocalStorageState('local-storage-form-data')
│   │       ├── 读取: localStorage → localStorageFormData[formKey]
│   │       └── 写入: setLocalStorageFormData(...)
│   │
│   ├── formData = omit(localStorageFormData[formKey], omitStorageKeys)
│   │   排除: prompt, enableMagicPrompt, loras(条件), published, protectionMode
│   │
│   ├── ProForm.useWatch(() => syncFormData(), form)
│   │   └── 表单变化 → form.getFieldsValue() → omit → 写回 localStorage
│   │
│   └── 返回: { formData, formDataInitialValues, loadFormDataFromLocalStorage }
│
├── useFormRemoteValues({ postRemoteData, getFieldsValue, setFieldsValue })
│   │
│   ├── useInflowRouteQueryStoreSelector(...)
│   │   └── 从 Context store 读取: recordId, videoId, styleId, lora, modelName, ...
│   │
│   ├── initialType 推导:
│   │   recordId  → 'generating'
│   │   videoId   → 'similar'
│   │   customType→ 'custom-xxx'
│   │   lora      → 'lora'
│   │   否则      → null
│   │
│   ├── usePrivacySettings()
│   │   └── localStorage['privacy-settings-{userId}'] → setFieldsValue
│   │
│   ├── 4 个 tRPC useQuery（互斥，enabled 控制只发 1 个）
│   │   └── API 返回 → handleRemoteInitialValues → setFieldsValue
│   │
│   ├── styleId / modelName → 直接 setFieldsValue
│   │
│   └── 返回: { initialType, loading, initialed, generationId }
│
└── 最终返回: { formData, loadFormDataFromLocalStorage, initialType, loading, ... }
```

## 四、useFormRemoteValues 4 个 API 请求对比

| 触发条件 | `initialType` | API 调用 | 入参 | 出参用途 |
|---------|--------------|----------|------|---------|
| `?recordId=123` | `'generating'` | `video.getGeneratingVideo` | `{ id: 123 }` | 恢复生成中的任务配置 |
| `?videoId=456` | `'similar'` | `video.getVideoDetail` | `{ videoId: '456' }` | 一键同款，复制作品配置 |
| `?recordId=789&customType=reprompt` | `'custom-reprompt'` | `generation.queryRecordDetail` | `{ id: 789 }` | 失败重试，重新拉取配置 |
| `?lora=uuid` | `'lora'` | `imageLora.detail` | `{ uuid: 'xxx' }` | 从 LoRA 详情页跳转，选中该 LoRA |

### 不走 API 的直接回填

| URL 参数 | 处理方式 |
|---------|---------|
| `?styleId=xxx` | `setFieldsValue({ style: styleId })` |
| `?modelName=flux` | `setFieldsValue({ modelName })` |
| `?type=video-to-video` | `setFieldsValue({ video: { src, metadata } })` |
| `?type=lip-sync` | `setFieldsValue({ video: { src, metadata } })` + 唇形校验 |
| `?type=image-to-image` | `setFieldsValue({ imageUrl, images })` |

### initialType vs state.query.type

| | `initialType` | `state.query.type` |
|---|---|---|
| 来源 | hook 内部根据多个参数推导 | URL `?type=xxx` 直接传入 |
| 决定什么 | **调哪个 API** | **怎么回填表单**（回调内部分支） |
| 可能的值 | `generating` / `similar` / `custom-xxx` / `lora` / `null` | `video-to-video` / `lip-sync` / `image-to-image` / `undefined` |
| 执行阶段 | 先判断，决定发哪个请求 | API 返回后，决定 setFieldsValue 的内容 |

## 五、附：URL query 参数获取链路

> `useFormRemoteValues` 不直接用 `useRouter`，而是通过 Zustand + Context 读取 query 参数，
> 避免 `useRouter` 在任意 query 变化时导致全组件树重渲染。

```
用户访问 /ai-image-generator?videoId=123&modelName=flux
│
▼
Next.js router.query = { videoId: '123', modelName: 'flux', ... }
│
▼
InflowRouteQueryProvider（组件挂载时，一次性读取 router.query）
│
├── const { projectId, host, ...restQuery } = router.query
│   排除内部参数 projectId、host
│
├── createInflowRouteQueryStore({ query: restQuery })
│   创建 Zustand store 实例，存入 { videoId: '123', modelName: 'flux' }
│
└── <InflowRouteQueryStoreContext.Provider value={store}>
    通过 React Context 下发 store 引用
    │
    ▼
    useInflowRouteQueryStoreSelector((state) => ({
      recordId:  state.query?.recordId,   // undefined
      videoId:   state.query?.videoId,    // '123'
      modelName: state.query?.modelName,  // 'flux'
      styleId:   state.query?.styleId,    // undefined
      lora:      state.query?.lora,       // undefined
      ...
    }))
    使用 shallow 比较，只在选中字段变化时重渲染
    │
    ▼
    useFormRemoteValues 拿到参数，开始调度 API
```

### 组件层级关系

```
InflowRouteQueryProvider          ← 读取 router.query → 创建 store → Context 下发
  └── CreationLayout
        └── FormGenerate
              └── useFormInitialValues
                    └── useFormRemoteValues
                          └── useInflowRouteQueryStoreSelector  ← 从 Context 消费
```

## 六、附：模型落地页跳转链路

> 模型落地页（如 `/m/kling-ai`）不直接传表单数据到功能页，
> 而是通过 **localStorage 中转 + URL 信号参数** 实现跨页面表单传递。

```
模型落地页 /m/kling-ai
│
│  用户填写 prompt、选择图片、选择模型
│
├── 1. setFormValues(values)
│      写入 localStorage['by-model-page-local-form-values']
│      存储内容: { prompt, imageUrl, modelName, ... }
│
├── 2. router.push('/image-to-video?customType=modelPage')
│                                   ↑
│                   CreationFormCustomType.ModelPage = 'modelPage'
│                   这只是一个信号参数，不携带表单数据
│
▼
功能页 /image-to-video?customType=modelPage
│
├── useFormRemoteValues
│   ├── initialType = 'custom-modelPage'
│   └── 4 个 API 的 enabled 全为 false → 不发任何请求
│
├── useLocalFormValuesFromModelPage({ form, initialType })
│   ├── 检测 initialType === 'custom-modelPage' ✓
│   ├── 读取 localStorage['by-model-page-local-form-values']
│   ├── 格式化（处理多图模型的 imageUrl → images 转换）
│   └── form.setFieldsValue(values) → 表单回填完成
│
▼
用户看到功能页，表单已自动填好模型落地页的配置
```

### ③ useFormRemoteValues 与 ⑤ useLocalFormValuesFromModelPage 互斥关系

| URL 参数 | ③ 远程 API | ⑤ 模型落地页 localStorage |
|---------|:---:|:---:|
| `?videoId=123` | 发请求，回填 | 不生效 |
| `?recordId=456` | 发请求，回填 | 不生效 |
| `?lora=uuid` | 发请求，回填 | 不生效 |
| `?customType=modelPage` | 不发请求 | 从 localStorage 回填 |
| 无参数 | 不发请求 | 不生效 |
