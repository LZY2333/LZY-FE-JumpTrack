# figma mcp ui skills

custom-figma Skill（上层工作流）的存在意义:

根据 skill 描述，它做的是：

1. 解析 Figma 链接，调用 MCP 获取设计稿数据
2. 对比现有代码组件（读取项目中已有的组件实现）
3. 输出改造分析文档（diff 格式，告诉你哪里要改、怎么改）

关系总结

┌──────────────────┬──────────────────┬──────────────────────┐
│ │ Figma MCP │ custom-figma Skill │
├──────────────────┼──────────────────┼──────────────────────┤
│ 层级 │ 底层数据获取 │ 上层分析工作流 │
├──────────────────┼──────────────────┼──────────────────────┤
│ 输入 │ fileKey / nodeId │ Figma 链接 │
├──────────────────┼──────────────────┼──────────────────────┤
│ 输出 │ 原始 JSON 数据 │ 代码改造方案（diff） │
├──────────────────┼──────────────────┼──────────────────────┤
│ 是否理解项目代码 │ 否 │ 是，会对比现有组件 │
├──────────────────┼──────────────────┼──────────────────────┤
│ 依赖关系 │ 独立 │ 内部会调用 MCP │
└──────────────────┴──────────────────┴──────────────────────┘

简单说：MCP 是手，Skill 是脑。没有 Skill，你拿到 Figma 数据后还得自己分析怎么改代码；有了 Skill，它帮你把「设计稿 → 代码改动方案」这个流程标准化了。
