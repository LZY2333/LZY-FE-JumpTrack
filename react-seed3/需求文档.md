# 需求文档

## 1 特性概述

目前投资移民业务后台处理流程包括年费收取、客户投资行为监控、申报资产至政府机构等，其中客户投资行为监控部分单日处理的业务量最大。而流程中存在大量手工操作，业务处理效率较低，难以支持前线业务的发展。现需要加强系统自动化处理业务的能力，缩短业务处理时长，以应对业务量的增长。

## 3 特性全景

### 3.1 特性全景描述[必需]

**综合查询系统监控CIES账户变动**

账户运营系统每天（T+1）监控客户的CIES账户T是否有扣账/入账。

**综合查询系统生成CIES结单**

系统如监控到存款账户发生变动，T生成存款账户结单及交易流水明细，并将结单存放至指定位置。

系统如监控到存款账户存在资金转出，将结单附加cover letter推送至OPC AET Maker（CIES2.0）。

**OPC AET Maker核查及维护申报内容**

OPC AET Maker T+1收到系统推送的CIES结单，核查结单是否完整，并按需更改邮件内容/上传附件后点击Pass；如不需申报，点击Cancel结束任务，任务不流转至Checker。

**OPC AET Checker 复核Maker提交的任务**

OPC AET Checker收到CIES结单及Maker录入的邮件内容，再次核查后，如无误点击Pass，如有误则点击Return，任务回到Maker节点。Pass成功后自动发送电邮通知CIES小组，并将结单及cover letter添加至附件。

## 4 特性说明

### 4.1 功能说明[必需]

#### 系统监控CIES账户变动

此功能已于CTS系统建设，迁移至BCL系统中CIES模块：

综合查询系统每天（T+1）监控客户的存款账户T是否有出账/入账（包括利息收入），将监控到的所有账户变动生成的结单及明细存至系统指定路径，并显示客户变动资产的时间（T）于目录。（存款账户为带有Hold Code D98的所有账户，出账/入账为账户流水发生的存款及提款）

不符合要求的变动，由系统生成Cover letter，附加存款日结单推送至CIES Maker任务池。（不符合要求的变动即账户流水发生的提款）

| 文件类型       | 附件内容                                                                                                                                                                                                      |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Cover Letter   | Cover Letter为PDF，不可修改，Our Ref、Your Ref、AIO Account同步系统记录的客户资料，Cover Letter日期同步Checker提交任务的日期/同步Today至任务结束，AIO Account Statement的日期同步发生交易的日期。即结单日期。 |
| 存款账户日结单 | 结单为PDF，不可修改，结单包括所有账户内交易流水。 Fund Dividend/Cash (派息/现金) 理财 （卖出/买入/转账） 汇款（转出/转入） Interest（存款利息）等存款账户发生的交易流水                                       |
| 交易流水       |                                                                                                                                                                                                               |

#### OPC AET Maker核查结单及录入资料

OPC AET Maker T收到系统推送的结单以及cover letter，核查客户资料及报告内容，比对客户的基本信息，核实T交易的总数是否正确，如无误则点击确认提交。

如Maker检查出结单有误（报告包含非CIES账户信息/客户信息有误/交易信息缺失等），由IT重新发起跑批任务，及取消已生成有误结单的任务。如系统未能按时重新生成正确的报告，Maker使用原流程处理。

#### OPC AET Checker 复核结单及资料

Maker提交任务后，OPC AET Checker收到合并CIES2.0结单及cover letter后的PDF文档，界面高亮Maker修改过的字段。再次核查后点击确认提交，系统将核查无误的结单及cover letter发送电邮至OPC CIES小组。将审核无误的结单保留至客户终止委托后10年。
