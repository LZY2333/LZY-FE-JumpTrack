## 1 特性概述

### 1.1 背景

目前投资移民业务后台处理流程包括年费收取、客户投资行为监控、申报资产至政府机构等，其中**客户投资行为监控**部分单日处理的业务量最大。而流程中存在大量手工操作，业务处理效率较低，难以支持前线业务的发展。

### 1.2 目标

加强系统自动化处理业务的能力，缩短业务处理时长，以应对业务量的增长。

### 1.3 范围

本需求覆盖 CIES（Capital Investment Entrant Scheme）账户变动的自动监控、结单生成、及 OPC 双人复核（Maker-Checker）流程的系统化改造，功能从 CTS 系统迁移至 BCL 系统的 CIES 模块。

---

## 2 关联专题衡量指标

| 指标 | 现状 | 目标 |
| ----- | ----- | ----- |
| 单笔任务平均处理时长 | 人工处理，耗时较长 | 系统自动生成结单，Maker 仅需核查与提交 |
| 日处理任务量上限 | 受人工操作瓶颈限制 | 支持业务量增长，减少人工干预环节 |
| 结单生成准确率 | 依赖人工，存在遗漏风险 | 系统自动监控并生成，减少人为错误 |
| 合规申报及时率 | 依赖人工跟进 | 系统推送 + 到期提醒，确保 T+1 申报 |

## 3 特性全景

### 3.1 特性全景描述

#### 综合查询系统监控 CIES 账户变动

账户运营系统每天（**T+1**）自动监控客户 CIES 账户在 **T 日**是否有扣账 / 入账。

#### 综合查询系统生成 CIES 结单

- 系统如监控到存款账户发生变动，在 T 日生成**存款账户结单**及**交易流水明细**，并将结单存放至指定路径。
- 系统如监控到存款账户存在**资金转出**，将结单附加 Cover Letter 推送至 **OPC AET Maker 任务池**（CIES 2.0）。

#### OPC AET Maker 核查及维护申报内容

OPC AET Maker 在 T+1 收到系统推送的 CIES 结单，核查结单完整性，并按需修改邮件内容 / 上传附件后点击 **Pass** 提交；如不需申报，点击 **Cancel** 结束任务（任务不流转至 Checker）。

#### OPC AET Checker 复核 Maker 提交的任务

OPC AET Checker 收到 CIES 结单及 Maker 录入的邮件内容，再次核查后：

- 无误 → 点击 **Pass**，系统自动发送电邮通知 CIES 小组，并将结单及 Cover Letter 作为附件。
- 有误 → 点击 **Return**，任务回退至 Maker 节点。

### 3.2 业务流程图

```txt
系统（T+1）监控 CIES 账户
        │
        ├── 无变动 → 结单存至指定路径（不推送任务）
        │
        └── 存在资金转出
                │
                ▼
        生成 Cover Letter + 日结单
                │
                ▼
        推送至 Maker 任务池
                │
                ▼
        Maker 核查 & 录入资料
                │
                ├── 结单有误 → 通知 IT 重跑批次 / 使用原流程
                ├── Cancel → 任务结束
                └── Submit → 流转至 Checker
                                │
                                ├── Return → 回退至 Maker
                                └── Approve → 发送电邮 → 任务归档
```

## 4 特性说明

### 4.1 系统监控 CIES 账户变动

此功能已于 CTS 系统建设，迁移至 BCL 系统 CIES 模块：

综合查询系统每天（T+1）监控客户存款账户（带有 **Hold Code D98** 的所有账户）在 T 日是否有出账 / 入账（包括利息收入）。

- 将监控到的所有账户变动，生成结单及明细，存至**系统指定路径**，并在目录中显示客户变动资产的日期（T）。
- **不符合要求的变动**（即账户流水中发生的**提款**），由系统生成 Cover Letter，附加存款日结单推送至 CIES Maker 任务池。

#### 系统生成附件规格

| 文件类型 | 格式 | 内容说明 |
| --------- | ----- | --------- |
| Cover Letter | PDF，不可修改 | Our Ref、Your Ref、AIO Account 同步系统记录；Cover Letter 日期同步 Checker 提交任务的日期（任务未结束时同步 Today）；AIO Account Statement 日期同步发生交易日期（即结单日期） |
| 存款账户日结单 | PDF，不可修改 | 包含所有账户内交易流水：Fund Dividend/Cash（派息/现金）、理财（卖出/买入/转账）、汇款（转出/转入）、Interest（存款利息）等 |
| 交易流水明细 | 系统生成 | 账户流水详细记录，附于结单之后 |

---

### 4.2 OPC AET Maker 核查结单及录入资料

OPC AET Maker 在 T+1 收到系统推送的结单及 Cover Letter，完成以下操作：

1. 核查客户基本信息（姓名、账号、客户类别等）
2. 比对 T 日交易总数是否正确
3. 按需修改可编辑字段、上传附件
4. 点击 **Submit** 提交任务至 Checker，或点击 **Cancel** 结束任务

**异常处理：** 如 Maker 检查出结单有误（包含非 CIES 账户信息 / 客户信息有误 / 交易信息缺失等），通知 IT 重新发起跑批任务，并取消已生成的有误结单任务。如系统未能按时重新生成正确报告，Maker 使用原有流程处理。

---

### 4.3 OPC AET Checker 复核结单及资料

Maker 提交任务后，OPC AET Checker 收到合并后的 PDF 文档（CIES 2.0 结单 + Cover Letter），界面**高亮显示 Maker 修改过的字段**。

Checker 完成复核后：

- 无误 → 点击 **Approve**，系统自动发送电邮，任务结束，处理记录保存至查询菜单
- 有误 → 点击 **Return**，任务回退至 Maker 节点

审核无误的结单须**保留至客户终止委托后 10 年**。

#### 电邮发送规格

| 电邮类型 | 收件地址 | 附件命名规则 | 邮件正文 / 备注 |
| --------- | --------- | ------------- | ---------------- |
| 报送投资推广署 | <wmdcies@cmbwinglungbank.com> | `{Your Ref}_{Our Ref}_Statement_{Date}` | Dear CIES Team: Please report to HK Invest by today as attached. 密码：由 OPC 设定 |
| 通知客户 | <wmdcies@cmbwinglungbank.com> | `{Your Ref}_{Our Ref}_Statement_{Date}` | 密码规则：姓氏拼音首字母大写 + 其余字母小写 + 出生月日年（MMDDYYYY）+ `!@`，例：`Chen12012024!@` |

> `Date` 均取 Checker 提交任务的日期。

#### 逾期提醒机制

任务发起后 **5 个工作日内**，如 Checker 未完成提交，系统在 Maker 及 Checker 界面弹出提示：

> **Tasks due at 2 days.**  
> 列出即将到期的任务 Reference No，用户点击确认后关闭弹窗。

#### 界面元素（第二节点 – OPC ASS Checker）

Checker 界面与 Maker 界面一致，但**所有字段均不可修改**。

#### 界面功能（第二节点 – OPC ASS Checker）

| 功能 | 界面元素 | 说明 |
| ----- | --------- | ----- |
| 打印 | Print | 打印指定页或全部页 |
| 下载 | Download | 下载保存至 WS 生产机 |
| 回退 | Return | 任务回退至 Maker 节点 |
| 提交 | Approve | 自动发送电邮，任务结束，处理记录保存至查询菜单 |

## 5 系统界面设计

### 5.1 系统主界面

系统主界面分为三个主要区域：

| 区域 | 说明 |
| ----- | ----- |
| Maker / Checker 操作区 | 当前任务详情及可操作按钮 |
| Task Pool | 待处理任务列表，双击进入处理界面 |
| Attachment | 附件预览及管理区域 |

### 5.2 主菜单

| 菜单功能 | 说明 |
| --------- | ----- |
| Task Pool | 展示系统生成的违规变动任务列表；仅已配置权限的 Maker / Checker 可操作对应角色任务；**同一笔任务的 Maker 与 Checker 不能为同一人** |

### 5.3 权限控制

| 角色 | 权限说明 |
| ------ | ---------- |
| OPC AET Maker | 可查看、编辑、提交、取消任务；不可 Approve |
| OPC AET Checker | 可查看、下载、打印任务；可 Return 或 Approve；不可修改任何字段 |
| 系统管理员 | 配置 Maker / Checker 权限，管理用户角色 |

## 6 非功能需求

| 类别 | 要求 |
| ------ | ------ |
| 数据保留 | 审核通过的结单保留至客户终止委托后 10 年 |
| 文件格式 | Cover Letter 及存款账户日结单均为 PDF，系统生成后不可修改 |
| 时效性 | 系统在 T+1 完成账户变动监控及结单推送 |
| 合规性 | 符合 CIES 投资移民业务监管要求，申报记录可追溯 |
| 安全性 | 对外发送的电邮附件须设置密码保护 |
| 可用性 | Task Pool 支持到期提醒，防止任务超时未处理 |
