# 首页渲染链路

## 组件嵌套树

```
_app.page.tsx                                    ← Next.js 应用入口，处理 getInitialProps 获取 hostname/locale/翻译等数据
│
├── DynamicGlobalComponent                       ← 动态加载的全局组件（如全局弹窗等）
├── DynamicGlobalComponentForDev                 ← 开发环境专用的调试组件
├── FaroObservability                            ← Grafana Faro 前端可观测性（错误/性能上报）
├── CoreDataScript                               ← 注入核心数据脚本（如用户信息、配置等）
├── ABTestScript                                 ← AB 测试分组脚本
├── PageWaitingScript                            ← 页面加载等待脚本（骨架屏/loading 控制）
├── UserPreferenceScript                         ← 用户偏好设置脚本（主题、语言等）
│
└── SiteContextProvider                          ← 站点上下文：hostname、hostKey、站点配置、多语言媒体资源
    └── AppSessionProvider                       ← 认证会话：包裹 AbTestProvider + NextAuth SessionProvider + 匿名登录
        └── ErrorBoundary                        ← 全局错误边界，捕获 React 渲染异常并展示降级 UI
            └── AppUIModalProvider               ← 全局 UI 弹窗状态管理（登录弹窗、支付弹窗、举报弹窗等）
                └── I18N                         ← Lingui 国际化 Provider，加载翻译资源
                    │
                    │  ▼ Component = SuperJsonPage（withSuperJSONPage 包裹后的页面）
                    │  ▼ withLayouts 从数组末尾向前依次包裹，最后一项在最外层
                    │
                    └── SuperJsonPage            ← SuperJSON 序列化支持（处理 Date/Map 等复杂类型的 SSR 传输）
                        │
                        │  ── layoutList（3 层，最外） ──
                        │
                        ├── SiteSpecificContextLayout          ← 站点业务上下文 + 免费奖励积分初始化
                        │   └── SiteSpecificContextProvider    ← 站点级业务状态（弹窗开关、视频操作、子应用标识等）
                        │
                        ├── FormStoreProvider                  ← 创作表单全局 store（styles/模型参数等），zustand Context 模式
                        │
                        ├── AntdLayout                         ← Ant Design ConfigProvider + 主题配置
                        │
                        │  ── RootLayout（第 2 层） ──
                        │
                        ├── RootLayout                         ← 站点根布局：主题样式注入、SEO <Head>、签到通知、AB 测试
                        │   └── InflowRouteQueryProvider       ← 信息流页面路由查询参数管理（避免 useRouter 导致的重渲染）
                        │       └── BaseLayout                 ← 基础 HTML 布局：<Head> SEO 标签、页面切换进度条
                        │           └── NuqsAdapter            ← nuqs URL 状态管理适配器（Pages Router 版本）
                        │               └── MediaToolsProvider ← 媒体工具弹窗状态（抠图、去背景等工具的弹窗开关）
                        │
                        │  ── LayoutsDefaultLayout（第 1 层，最内） ──
                        │
                        ├── SitePromotionPopupsProvider         ← 站点促销弹窗管理（优先级排序、开关控制）
                        │   └── DefaultLayout                   ← 页面 UI 骨架：Header + main + Footer + FooterTab
                        │       ├── Header                      ← 顶部导航栏（logo、菜单、用户信息、语言切换）
                        │       ├── <main>                      ← 页面主内容区域（children）
                        │       ├── Footer                      ← 底部页脚（链接、社交媒体、版权信息）
                        │       └── FooterTab                   ← 移动端底部固定标签栏
                        │
                        │  ── Page（首页业务组件） ──
                        │
                        └── Page
                            └── MenuLayout                     ← 菜单布局容器
                                ├── BannerCarousel             ← 首页轮播 Banner（运营位）
                                ├── ChatCreateFlowProvider     ← 聊天创作流程上下文
                                │   └── ChatBoxWithTab         ← 带标签切换的 AI 对话框（桌面端）
                                ├── Brand                      ← 品牌展示区（移动端）
                                ├── MasonryGridList             ← 瀑布流内容列表（按分类展示作品）
                                ├── Faq                        ← 常见问题
                                └── ExploreMore                ← "探索更多"入口
```

## 全局 Provider 一览

以下 Provider 按嵌套层级从外到内排列，标注其**持久性**（路由切换时是否销毁重建）。

### 持久层（挂载在 `_app.page.tsx`，路由切换不销毁）

| Provider | 文件 | 功能 |
|---|---|---|
| **SiteContextProvider** | `web/src/contexts/SiteContext/` | 提供 hostname、hostKey、站点运行时配置（GTM ID、支付配置等）、多语言媒体资源 |
| **AppSessionProvider** | `web/src/_blocks/AppSessionProvider/` | 包裹 `AbTestProvider`（AB 测试分组）+ NextAuth `SessionProvider`（用户会话）+ 匿名自动登录 + 数据埋点初始化 |
| **ErrorBoundary** | `web/src/components/ErrorBoundary/` | React 错误边界，捕获子树渲染异常，展示降级 UI，路由切换时自动 reset |
| **AppUIModalProvider** | `web/src/contexts/AppUIModalContext/` | 全局 UI 弹窗控制器：登录/注册弹窗、支付弹窗、举报弹窗、视频详情弹窗等的 open/close 状态 |
| **I18N** | `web/src/components/I18N/` | Lingui `I18nProvider` 包裹，加载当前 locale 对应的翻译文件 |

### 页面层（挂载在 withLayouts 链路，路由切换时销毁重建）

| Provider | 文件 | 功能 |
|---|---|---|
| **SiteSpecificContextProvider** | `web/src/contexts/SiteSpecificContext/` | 站点级业务状态：各类弹窗开关（发布视频、举报、Lora 详情等）、子应用标识、免费奖励积分初始化 |
| **FormStoreProvider** | `web/src/pages/pollo.ai/_context/Form/` | 创作表单 zustand store：风格列表、模型参数等表单初始状态，通过 Context 传递 store 实例 |
| **AntdLayout** | `web/src/pages/pollo.ai/_layout/layouts/AntdLayout/` | Ant Design `ConfigProvider` 主题配置 |
| **InflowRouteQueryProvider** | `web/src/pages/pollo.ai/_context/InflowQuery/` | 信息流路由查询参数的 zustand store，替代 `useRouter().query` 避免不必要的重渲染 |
| **NuqsAdapter** | 第三方库 `nuqs` | URL query 参数状态管理的 Pages Router 适配层 |
| **MediaToolsProvider** | `web/src/pages/pollo.ai/_hooks/useMediaTools/` | 媒体编辑工具弹窗状态（去背景、裁剪等工具的弹窗 open/close 和当前工具类型） |
| **SitePromotionPopupsProvider** | `web/src/contexts/SitePromotionProvider/popups/` | 促销弹窗管理：按优先级排序、控制同一时间只展示一个促销弹窗 |

## 路由切换时的生命周期

```
_app.page.tsx（不销毁）
├── SiteContextProvider        ✅ 持久
├── AppSessionProvider         ✅ 持久
├── ErrorBoundary              ✅ 持久
├── AppUIModalProvider         ✅ 持久
├── I18N                       ✅ 持久
│
└── Component ← 路由切换时整体替换
    ├── SiteSpecificContextLayout  ❌ 销毁重建
    ├── FormStoreProvider          ❌ 销毁重建
    ├── AntdLayout                 ❌ 销毁重建
    ├── RootLayout                 ❌ 销毁重建
    ├── DefaultLayout              ❌ 销毁重建
    └── Page                       ❌ 销毁重建
```

Next.js Pages Router 中，`_app.page.tsx` 的 `Component` prop 在路由切换时会替换为新页面组件。由于 `Component` 及其 key 发生变化，React 会卸载整棵旧子树并挂载新子树。因此：

- **持久层**的 Provider 状态在路由间保持（用户登录态、站点配置、国际化等）
- **页面层**的所有 Provider 和 zustand store 实例会随路由切换而**销毁重建**，不存在跨路由数据缓存
