# 生成需求文档 skill

AI-ready Requirement Document

需求文档(产品) -> AI能理解的需求文档(开发) -> 详设文档plan(AI） -> 详设评审(开发) -> 编码(AI) -> Review(开发)

## skill背景介绍

这个 Skill 本质上是一个 需求文档 → AI-Spec 转换器，目标是让 AI 能：

理解需求

生成详设

写伪代码

写代码

所以 Skill 的设计重点不是“写文档”，而是 让 AI 能稳定理解需求结构。
