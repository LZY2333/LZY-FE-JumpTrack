# Skill 编写要求

写新 skill、改老 skill 时对照本清单。读者默认是能力较弱的 AI。

1. **使用正向祈使句** —— 用"做什么"的命令式语气,不用"可以""建议""一般来说"这类含糊或被动表述。

2. **使用精准的 Step,减少解释** —— 步骤本身说清动作,少用 `()` `>` 之类的补充注解去解释。

3. **要使用 Demo 模板** —— 必须给成品 Demo,让执行者照着格式产出,而不是只给抽象描述。

4. **明确使用的工具** —— 每步点名具体工具(Glob / Grep / Read / Edit / Write 等),不写"查一下""打开"这种模糊动作。

5. **给边界** —— 写清这一步只做什么、不碰什么,划定范围防止越界。

6. **给判断标准** —— 给出可验证的判据(自检清单、可 Grep 的 token、判定分支),让执行者能自己判断对错,而不是凭感觉。
