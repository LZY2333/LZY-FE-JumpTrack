# 注释提示语规则（代码注释）

## 适用场景

当用户要求"给某个文件加注释"，且要求注释对象包含：

- 自定义函数
- 自定义变量
- 解构的变量
- 自定义 React Hooks
- JSX 内每个自定义组件
  并且注释内容是"注释对象的作用/功能"。

## 产出原则

- 无需二次确认，直接写入文件（覆盖全局代码改动审批流程）。
- 用简洁中文说明"做什么/用途是什么"。
- 不改变代码逻辑，仅新增注释。
- 如果代码中已有注释，保留原注释，不重复添加。
- JSX 内注释必须使用 `{/* ... */}`；TS/JS 使用 `//`。
- 若 JSX 顶层不能直接插入注释，将注释上移至JSX外。

## 注释覆盖清单

1. **自定义函数**：在函数定义前加 `//` 说明职责。
2. **自定义变量**：在变量声明上方加 `//` 说明用途。
3. **解构变量**：在解构对象内为每个字段加行内注释说明用途。
4. **自定义 Hooks**：在 `useXxx()` 调用上方加 `//` 说明该 hook 的作用（如 `// 获取路由信息`、`// 管理弹窗显隐状态`）。
5. **JSX 自定义组件**：在组件标签前添加 `{/* ... */}`，需包含：
    - **展示位置**：组件在页面中的位置（如"顶部导航栏右侧"、"表单底部"）。
    - **展示效果**：组件呈现的 UI 效果（如"显示用户头像下拉菜单"、"渲染积分余额标签"）。
    - **展示时机**（仅条件渲染时）：触发展示的条件（如"用户已登录时展示"、"移动端隐藏"）。
6. **对象字面量 key**：在返回或构造的大对象中，为每个 key 添加注释说明用途。

## 代码结构图

注释完成后，在文件底部以 `/* ... */` 注释块生成当前文件的树状结构图。

### 规则

- 自定义组件直接写组件名。
- 兄弟节点是原生 HTML 元素时，写中文描述（如 `移动端汉堡菜单图标`、`页面标题区域`）。
- 每个节点后加中文注释说明其作用。
- 条件渲染的节点用 `（条件：xxx）` 标注。

### 示例

```tsx
/*
 * 代码结构图：
 *
 * Header
 * ├── TopBanners                    — 顶部活动横幅
 * ├── <header>
 * │   ├── 左侧
 * │   │   ├── Logo                  — 网站 Logo，点击回首页
 * │   │   └── 移动端汉堡菜单图标
 * │   └── 右侧
 * │       ├── HeaderMenu            — 顶部导航菜单
 * │       ├── CreditsInfo           — 积分余额展示
 * │       └── LoginUserInfo         — 登录用户头像/下拉菜单（条件：已登录）
 * └── FaroObservability             — 前端可观测性上报
 */
```

## ASCII UI 布局图

当目标文件包含 JSX 代码块时，在"代码结构图"之后再追加一个 `/* ... */` 注释块，绘制 ASCII UI 布局图。

### 判断条件

文件内包含 JSX 代码块（`return (` 内含 JSX 标签，或导出 JSX 表达式）即生效。
纯 hooks、工具函数、类型文件、常量文件跳过。

### 示例

```tsx
/*
 * ASCII UI 布局图：
 *
 * ┌─────────────────────────── Header ───────────────────────────┐
 * │ ┌─────────────────── TopBanners ──────────────────────────┐  │
 * │ │              顶部活动横幅 (可关闭)                        │  │
 * │ └────────────────────────────────────────────────────────┘  │
 * │ ┌──────────────────── <header> ───────────────────────────┐  │
 * │ │ ┌──── 左侧 ─────┐          ┌──────── 右侧 ──────────┐ │  │
 * │ │ │ Logo  │ ☰ 菜单 │          │ Menu │ Credits │ Avatar │ │  │
 * │ │ └────────────────┘          └─────────────────────────┘ │  │
 * │ └────────────────────────────────────────────────────────┘  │
 * └─────────────────────────────────────────────────────────────┘
 */
```

## 输出格式

- 直接修改原文件，保持现有代码风格与格式。
- 不新增无关注释，不引入额外逻辑。
