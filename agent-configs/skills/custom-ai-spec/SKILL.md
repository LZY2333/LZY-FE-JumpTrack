---

## 工作流

### 步骤 1：接收输入

- 如果用户在命令后**已带输入内容**（文字、截图、链接等），直接作为需求描述使用
- 如果**无输入**，输出以下建议后等待用户输入：

```
请描述你的需求，例如：
- 一句话需求："用户头像支持裁剪上传"
- 指定模块："修改 Header 组件的导航菜单"
- 贴设计稿截图
- 贴 PRD 链接或文档内容
```

### 步骤 2：分析需求 + 扫描代码

1. 解析用户输入，提取所有需求点（不得遗漏任何一条）
2. 如果用户提到了涉及模块，以此缩小检索范围；否则根据需求内容推断
3. 扫描项目代码，理解涉及模块的现有结构、文件、组件关系
4. 将分析结果结合下方模板，生成 `plan/0.AI-SPEC.md`

### 步骤 3：输出

- 在项目根目录创建 `plan/` 目录（如不存在）
- 生成 `plan/0.AI-SPEC.md`
- 输出完成提示，告知用户审核并修改后运行 `/ai-plan` 生成执行计划

```markdown
## 项目约定

- 接口入参必须使用 Zod schema 校验
- 用户可见文案必须使用 Lingui 国际化
- 组件样式使用 Tailwind + figma-theme 颜色变量
- 使用 CustomModal / CustomSelect 替代 antd 原生组件
- 使用 useMemoizedFn 替代 useCallback
- 禁用枚举，使用联合类型或 as const
```

---

## 写作规范

- 非技术内容使用中文
- 技术术语、代码标识符保留原文
- **不含执行细节**（怎么改、改哪行），只描述"做什么"
- 每个需求点必须可追溯到用户原始输入
- 不得遗漏用户输入中的任何需求点
- 「项目约定」段为固定模板，用户可修改本 skill 文件来定制
