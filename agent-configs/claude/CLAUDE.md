# 全局行为规范

## 必须 每次回复 若涉及文件 末尾附带涉及文件的绝对路径

- 格式：绝对路径 `path:line`, 默认行号`:1`
- 单独成行, 按出现顺序排列, 不携带其他内容, 不重复

## 文件路径必须单独成行

- 除了为标识区分而出现的文件名片段（如 `MavatarList/index`）外，所有文件路径禁止内联在句中，必须单独成行。

## Console 时序诊断法

use when："打 console 分析"、"console 诊断"，或同一个 UI bug 修改 2 次仍未解决时自动切换。

1. **埋 log**：在关键节点（事件监听、API 代理、React 生命周期）埋带 `performance.now()` + 关键状态的 log
2. **给用户复现步骤**，让用户粘贴控制台输出
3. **按时间戳分析**：谁先谁后、谁没执行、状态在哪步不对
4. 确认根因后再改代码，不要盲猜

## 代码改动审批流程

对任何代码文件的修改，必须先完成以下步骤，用户明确同意后才能执行写入：

1. **给出改动计划**：说明要改哪些文件、改动目的和思路。
2. **给出 diff block**：使用 ```diff 格式展示具体变更内容。
3. **等待用户确认**：用户回复同意后，才执行实际的文件修改操作。
